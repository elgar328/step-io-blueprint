# step-io-blueprint

A schema-analysis tool that turns the EXPRESS schemas behind STEP into the
**code-generation inputs** for [step-io](../step-io), a STEP/EXPRESS
reader-writer.

step-io's reader, writer, and model are generated by its `codegen` crate from a
schema-faithful description of the merged AP schemas. This tool produces that
description: it reads the EXPRESS `.exp` schemas and emits two faithful exports —
no inference, no hand-authored shape decisions, deterministic from the schemas.

## Quick start

```sh
cargo run --release -- universal_export   # → inferred/universal.toml
cargo run --release -- profile_export     # → profiles/<target>.toml
cargo run --release                       # prints usage
cargo test                                # unit tests
```

## Exports

| Command | Output | What it is |
|---|---|---|
| `universal_export` | `inferred/universal.toml` | The schema-faithful **union** of all AP schemas (entities, attributes, SELECTs, ENUMs) plus per-entity DERIVE facts. The input the step-io `codegen` generator reads to build the reader/model. |
| `profile_export` | `profiles/<target>.toml` | One **output profile** per curated target AP (latest IS edition): the legal entity set + ordered attributes for schema-conditioned writing. Targets: `ap203e2`, `ap214e3`, `ap242e2`. |

Both read the EXPRESS schemas (below). `universal_export` additionally reads the
frozen `inferred/corpus_usage.toml` to recover real multi-instance (complex)
entity combinations that declaration-only rules miss.

### corpus_usage.toml — no live corpus dependency

`universal_export` never scans a STEP corpus. Per-entity usage counts are frozen
in `inferred/corpus_usage.toml` and committed here, so the tool runs with **no
external dependency**. That file is produced by a separate corpus-scanning tool
and copied in only when the corpus itself changes (rare).

## Schemas

Six EXPRESS schemas are read as a union: AP203 (Ed 1–2), AP214 (Ed 3), and
AP242 (Ed 1–3). The authoritative free source is the ISO **SMRL** (STEP Module &
Resource Library); legacy non-modular schemas the SMRL does not carry come from
[MBx-IF](https://www.mbx-if.org/home/mbx/resources/express-schemas/) /
[STEPCode](https://github.com/stepcode/stepcode). Line endings are normalized to
LF; schema content is unchanged.

| file | edition | source |
|---|---|---|
| `ap203e1.exp` | AP203 Ed 1 | STEPCode |
| `ap203e2.exp` | AP203 Ed 2 | SMRL v12 |
| `ap214e3.exp` | AP214 Ed 3 | MBx-IF |
| `ap242e1.exp` | AP242 Ed 1 | MBx-IF |
| `ap242e2.exp` | AP242 Ed 2 | SMRL v8 |
| `ap242e3.exp` | AP242 Ed 3 (TS) | SMRL v9 |

Cached SMRL release zips (v4–v12) live in `schemas/smrl/`. Only mechanical-CAD
schemas are used; AP209/210/238/239/240, IFC, ISO 15926, and PDM domains are out
of scope.

These `.exp` files are third-party ISO 10303 (STEP) schemas, not covered by this
repo's license. Full provenance, entity counts, model (MIM/AIM), and
publication-tier (IS/TS) notes are in [`schemas/NOTICE.md`](schemas/NOTICE.md).

## Layout

```
src/
├── main.rs                  CLI dispatch (universal_export / profile_export)
├── express.rs               EXPRESS schema parser (.exp → Schema, incl. SUPERTYPE clauses)
└── infer/
    ├── refgraph.rs          schema-union reference graph (entity parents / abstract / attr conflicts)
    ├── export_common.rs     shared exporter helpers (ty repr, schema rank, redeclaration signal)
    ├── universal_export.rs  → inferred/universal.toml (codegen input)
    └── profile_export.rs    → profiles/<target>.toml (output profiles)
schemas/                     the six .exp schema files (+ smrl/ cached SMRL release zips, NOTICE.md)
inferred/                    frozen corpus_usage.toml (input) + universal.toml (output)
profiles/                    per-target output profiles
```
